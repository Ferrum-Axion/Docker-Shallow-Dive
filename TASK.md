
# Legacy application containerization

As part of DevOps team you are required to handle dev tasks for enabling development process. In cases such this, handling old application with new technology is a requirement. As such, one much learn to use old application and setup new environment with those old apps.

Setup ssh server in a container and connect from your ssh client to container with ssh.

### Pre Requisites:

- Linux based OS (Debian and/or Rocky)
- Docker
- Docker compose
- Ssh-server (obviously)
- git
- Will to finish project
- Popcorn and/or bamba (you can use nutella, but I am not taking any responsibilities for your diabetes)

### Task

- Setup all the pre-requirements
- Learn to run ssh-server from shell in daemon form
- Create repository to save and document this project steps:
    - `README.md`
    - `TASKS.md`
    - `CONTRIBUTORS.md`
    - `setup.sh`
    - `Dockerfile`
    - `docker-compose.yml`
- Run application in a way that it will work as stand-alone container as well as multi-container environment
- In case of `docker-compose` mount project directory into the container or containers
